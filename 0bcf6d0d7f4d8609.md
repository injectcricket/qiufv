# LinkVault — 技术文章聚合与导航系统

LinkVault 是一个面向开发者与技术研究人员的轻量级技术资源导航与文章聚合平台。该项目旨在解决技术信息分散、优质内容难以检索与归类的问题，通过将分散于各技术博客、文档站点与社区的文章链接进行结构化整理，提供统一的检索与浏览入口。

项目定位于中小型技术团队、开源项目维护者以及个人技术博主，帮助其快速建立内部知识库或公开的技术资源导航站。LinkVault 不依赖外部数据库，基于静态 Markdown 文件与索引机制运行，部署简单，维护成本低，适合作为技术文档站点的补充工具。

## 功能概览

- **多维度文章索引**：按照技术领域、发布时间、热度等维度对收录链接进行自动归类，支持标签筛选与全文检索。

- **结构化数据展示**：将每一篇文章的标题、摘要、来源域名与发布时间提取为结构化字段，以表格形式在列表页呈现。

- **批量导入与更新**：支持通过 CSV 或 JSON 格式批量导入文章链接，系统自动去重并更新索引文件。

- **静态站点生成**：内置轻量级模板引擎，可一键生成完整的静态 HTML 站点，便于部署至 Nginx、Apache 或对象存储服务。

- **自定义分类体系**：用户可根据自身技术栈创建自定义分类目录（如后端、前端、运维、人工智能），并将链接灵活归类。

- **访问统计与热度标记**：记录每篇文章的外部点击次数，并自动标记热门文章，辅助用户筛选高价值内容。

- **RSS 订阅输出**：为每个分类目录生成独立的 RSS 订阅源，方便团队内部分发与订阅。

- **备份与迁移工具**：提供命令行工具用于导出全量索引数据，支持在不同实例间进行数据迁移。

## 应用场景

**技术团队内部知识库建设**：研发团队可将日常阅读的技术博客、解决方案文档、故障排查记录通过 LinkVault 统一收录，按项目或技术领域分类，新成员入职时可快速了解团队所关注的技术方向与积累的实践经验。

**开源项目文档中心补充**：开源项目维护者可以将与项目相关的生态工具、社区教程、迁移指南等外部资源聚合至 LinkVault，作为官方文档的补充延伸，帮助用户更全面地理解项目使用场景。

**个人技术博客的阅读清单**：技术博主或专栏作者可使用 LinkVault 管理自己的阅读清单，将已读与待读文章分类标记，同时公开分享给读者，增强读者互动与技术讨论氛围。

**技术社区资源导航站**：技术社区运营方可基于 LinkVault 快速搭建所属领域的资源导航站，收录高质量的教程、视频课程、工具推荐，降低社区成员的信息筛选成本。

## 快速开始

以下命令将帮助您在本地快速启动 LinkVault 实例：

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault.git
cd linkvault

# 安装依赖（使用 pip 和 npm 分别安装后端与前端依赖）
pip install -r requirements.txt
npm install --prefix frontend

# 构建前端静态资源
npm run build --prefix frontend

# 初始化索引数据库并导入示例数据
python manage.py initdb
python manage.py import --file samples/links.json

# 启动开发服务器（默认监听 8000 端口）
python manage.py runserver
```

访问 http://localhost:8000 即可浏览已收录的文章列表。如需生成静态站点，请执行：

```bash
python manage.py build --output ./dist
```

生成的文件位于 `./dist` 目录，可直接部署至 Web 服务器。

## 贡献指南

1. 在 GitHub 上 Fork 本项目仓库，并克隆至本地开发环境。建议在 `dev` 分支上进行所有修改，保持主分支与上游同步。

2. 安装开发依赖：执行 `pip install -r requirements-dev.txt`（包含 pytest、black、flake8 等工具），并运行 `pre-commit install` 启用代码提交前检查钩子。

3. 编写或修改代码后，请确保通过全部单元测试：运行 `pytest tests/`，且测试覆盖率不低于 85%。对于新增功能，请同步编写相应的测试用例。

4. 提交前运行代码格式化与静态检查：`black app/` 和 `flake8 app/`，确保代码风格与项目规范保持一致。提交信息请使用语义化格式，如 `feat: add batch import feature` 或 `fix: resolve duplicate key issue`。

5. 提交 Pull Request 时请清晰描述变更内容、涉及的问题编号（如有）以及测试验证结果。所有 PR 需至少一位项目维护者审核通过后方可合并。

## 常见问题

**Q：导入大量链接时出现超时或内存不足，应如何优化？**

A：对于超过 5000 条的批量导入，建议使用命令行工具分批执行：`python manage.py import --file data.json --batch-size 200`。同时可调整 SQLite 缓存设置，在 `config.yaml` 中增加 `sqlite_cache_size: 10000` 以提升写入性能。若内存仍不足，可考虑使用 `--no-validate` 跳过元数据验证步骤。

**Q：生成的静态站点如何部署到 HTTPS 环境？**

A：LinkVault 生成的静态资源默认使用相对路径，兼容 HTTP 与 HTTPS。若需强制所有链接使用 HTTPS，请在构建时设置环境变量 `SITE_PROTOCOL=https`，或在 `config.yaml` 中配置 `site_url` 为完整 HTTPS 域名。部署至 CDN 时请确保将 `_redirects` 文件一并上传以支持 SPA 路由。

**Q：如何自定义页面主题或添加新模板？**

A：所有模板文件位于 `app/templates/` 目录，使用 Jinja2 语法。您可直接修改 `base.html` 中的 CSS 引用或布局结构。如需新增独立页面，可创建新模板并在 `app/services/page_generator.py` 中注册相应生成逻辑。修改后重新执行 `python manage.py build` 即可生效。
