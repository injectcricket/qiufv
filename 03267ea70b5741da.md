# LinkVault Resource Aggregator

LinkVault is a curated technical resource aggregation system designed for developers, researchers, and technical writers who need to organize, categorize, and rapidly access distributed web content. The project addresses the common problem of link rot, scattered bookmarks, and inefficient retrieval of reference materials by providing a structured indexing mechanism over a large corpus of external URLs.

Target users include backend engineers building knowledge-base systems, data analysts performing web resource audits, and technical documentation teams maintaining external reference lists. LinkVault does not host content but provides a robust metadata layer, verification tooling, and search interfaces over the indexed URL set.

## 功能概览

**URL Normalization Engine** - Automatically detects and normalises variant URL formats while preserving original strings for audit trails.

**Batch Import Pipeline** - Processes large URL lists in batches (current batch 110/280) with progress tracking and error logging.

**Content Fingerprinting** - Generates cryptographic hashes of target resources to detect content changes and prevent duplicate entries.

**Health Check Scheduler** - Periodically verifies link availability and response times, flagging broken or redirected endpoints.

**Tag Inference System** - Uses rule-based heuristics to suggest topical tags from URL structure and path components.

**Export Adapters** - Supports JSON, YAML, and Markdown table exports for integration with static site generators and documentation pipelines.

**Query Interface** - Provides filtering by domain, status code, last-checked timestamp, and custom metadata fields.

## 应用场景

**Technical Documentation Maintenance** - Documentation teams managing hundreds of external reference links can use LinkVault to automatically verify that all cited resources remain accessible before each release cycle. The batch import feature allows seamless ingestion of new reference lists from existing markdown files.

**Research Data Curation** - Researchers compiling literature or data source lists can import their URL collections, tag them by topic, and export structured metadata for inclusion in appendices or data management plans. The fingerprinting feature ensures version consistency across multiple collaborators.

**Legacy System Migration** - Organizations migrating from older content management systems can use LinkVault to audit external dependencies, identify broken links, and generate migration manifests that map old URLs to new endpoints.

**Compliance Auditing** - Compliance officers can schedule regular health checks on regulatory reference URLs, maintaining an auditable log of availability and content changes for internal and external reporting requirements.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/example/linkvault.git
cd linkvault

# Install dependencies
pip install -r requirements.txt

# Run the initial import with sample data
python linkvault.py import --batch 110 --source data/urls_batch_110.txt

# Start the health check service
python linkvault.py schedule --interval 86400

# Generate markdown report
python linkvault.py export --format markdown --output docs/references.md
```

## 贡献指南

1. 在 GitHub 上 fork 本仓库并克隆到本地开发环境。创建新的功能分支时使用 `feature/` 前缀，修复分支使用 `fix/` 前缀，确保分支命名语义清晰。

2. 安装开发依赖（`pip install -r requirements-dev.txt`）并配置 pre-commit 钩子以自动执行代码格式化和基本检查。所有提交必须通过 black 格式化、flake8 静态检查和 pytest 单元测试套件。

3. 为新功能编写对应的单元测试和集成测试，确保测试覆盖率达到 80% 以上。测试用例应放置在 `tests/` 目录下对应的子目录中，命名规范为 `test_*.py`。

4. 更新文档以反映任何用户可见的变更。如果新增了配置选项或命令行参数，须在 `docs/user-guide/` 中补充说明。API 变更须同步更新 `docs/api/` 中的 OpenAPI 规范。

5. 提交 pull request 时填写模板中的全部字段，包括变更摘要、测试结果、文档更新检查清单和关联 issue 编号。PR 需要至少一名核心维护者批准后方可合并。

## 常见问题

**Q: 导入大量 URL 时出现内存不足错误，如何解决？**

A: 对于超过 10000 条记录的批次，建议启用流式导入模式。使用 `--stream` 参数将单次提交拆分为每 500 条一批的事务，同时关闭内容指纹预计算（`--no-fingerprint`）以降低内存占用。若需生成指纹，可在导入完成后单独运行 `fingerprint --batch <id>` 命令。

**Q: 健康检查返回 403 或 429 状态码，如何配置重试策略？**

A: LinkVault 支持指数退避重试策略。在配置文件中设置 `retry.max_attempts: 5` 和 `retry.backoff_factor: 2.0`。对于已知会限流的域名，可在 `config/domain_overrides.yaml` 中为该域名单独配置 `rate_limit: 10`（每秒请求数）和 `retry_after: 60`（最小等待秒数）。

**Q: 如何从旧版 SQLite 数据库迁移到新版 schema？**

A: 使用内置迁移工具 `linkvault migrate --target-version latest`。该工具会自动检测当前数据库版本并顺序应用 `src/storage/migrations/` 目录下的迁移脚本。执行前请备份数据库文件。若迁移失败，可使用 `--rollback` 回退到上一个稳定版本。
