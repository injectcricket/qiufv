# TechIndex Resource Aggregator

TechIndex Resource Aggregator 是一个面向技术研究者、开发者和开源爱好者的外链资源汇总平台。项目定位于对分散在网络各处的优质技术文章、教程和文档进行系统化收录与分类导航，帮助用户快速定位所需的技术资料，避免在海量信息中反复检索。

本项目并非内容生产平台，而是一个精心维护的技术资源索引系统。通过人工筛选与自动化检测相结合的方式，对每一篇收录的文章进行主题分类、质量评估和时效性标记，确保索引库的准确性与可用性。目前索引库已收录超过两万条技术资源链接，涵盖编程语言、框架生态、算法结构、工程实践、运维监控等多个技术领域。第176批次共新增250个资源链接，均已通过可用性验证。

## 功能概览

**多维度分类导航**：按照技术领域、阅读难度、内容形式等多个维度对资源进行标签化分类，支持快速筛选。

**全文元数据检索**：基于标题、摘要、关键词和分类标签进行检索，支持布尔表达式和模糊匹配。

**每日增量更新**：资源库每日自动检测链接可用性，新增资源按批次发布，每批次均附带更新日志。

**RSS 订阅推送**：提供按分类和全量两种 RSS 订阅源，用户可订阅特定技术领域的资源更新。

**阅读状态追踪**：用户可标记已读、收藏或暂存资源，支持自定义标签和笔记备注。

**社区贡献入口**：开放资源提交通道，用户可通过 Pull Request 或 Issue 推荐优质资源。

## 应用场景

技术团队内部知识库建设：技术负责人可将本项目部署为团队内部的知识导航入口，新成员通过浏览分类索引快速了解团队所涉及的技术栈和常用参考资料，缩短上手周期。

个人开发者日常查阅：开发者在进行技术选型、框架学习或问题排查时，可通过本索引按关键词或分类快速定位到相关文章和教程，避免重复搜索和低效浏览。

开源项目文档补充：开源项目维护者可将本索引中的相关资源链接作为项目 README 或 Wiki 的扩展阅读部分，为使用者提供更丰富的学习材料和参考资料。

技术博客内容策划：技术博主和内容创作者可通过浏览本索引了解当前技术热点和常见话题分布，辅助策划选题和收集写作素材。

## 快速开始

以下指令适用于 Linux / macOS 系统，Windows 用户建议使用 WSL2 或 Git Bash 环境。

```bash
# 克隆仓库
git clone https://github.com/techindex/resource-aggregator.git
cd resource-aggregator

# 安装依赖（Python 3.10+ 环境）
pip install -r requirements.txt

# 初始化本地索引数据库
python scripts/init_db.py

# 启动本地 Web 服务（默认端口 8080）
python app.py --port 8080
```

访问 http://localhost:8080 即可进入本地索引浏览界面。生产环境部署请参考 `docs/deployment.md` 中的 Nginx + Gunicorn 配置示例。

## 贡献指南

欢迎社区贡献者参与本项目的资源收录和功能改进。所有贡献需遵守行为准则和贡献者许可协议。

第一步：浏览现有索引库，确认拟提交的资源未被收录。可通过检索功能或直接查阅分类导航进行查重。若发现重复提交，Pull Request 将被标记为无效。

第二步：在 GitHub Issues 中创建一个新议题，选择「资源推荐」模板，填写资源标题、原始链接、所属分类和推荐理由。若为功能改进或缺陷修复，请选择对应的议题模板。

第三步：Fork 本项目到个人账户，在本地创建特性分支（命名格式为 `feature/资源分类-简短描述` 或 `fix/问题编号`）。完成修改后提交 Pull Request 到主仓库的 `main` 分支。

第四步：等待核心维护者的代码审查和自动化测试通过。审查过程中可能会有修改意见，请及时响应。合并后，新增资源将在下一批次的更新日志中公示。

第五步：关注项目的 Release 页面和 RSS 订阅源，获取每次批量更新的详细内容和索引变更说明。活跃贡献者将被列入贡献者列表。

## 常见问题

问：资源链接失效了怎么办？

答：项目内置了每日自动检测机制，失效链接会被标记为不可用状态并移出主索引。用户若发现个别链接无法访问，可通过 Issues 提交「链接失效」报告，维护团队会在 24 小时内核实并处理。用户也可在本地部署时通过 `scripts/health_check.py` 手动触发全量检测。

问：我可以部署本项目用于私有团队的知识管理吗？

答：可以。本项目采用 MIT 许可证，允许自由使用、修改和再发布。私有部署时建议关闭 RSS 公网订阅和公开检索入口，并根据团队需求调整分类体系。我们提供了 `docs/operator/deployment-private.md` 专门说明私有化部署的配置要点。

问：如何获取本批次新增资源的详细元数据？

答：每批次发布时，`data/batch_manifest.json` 文件中会记录该批次所有资源的标题、摘要、分类标签和收录时间。用户可通过 Web 界面的「批次浏览」功能按批次号筛选查看，也可通过 API 端点 `/api/batch/{batch_id}` 获取结构化数据。本批次编号为 176，总批次计数为 280。
