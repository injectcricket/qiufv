# Blog Navigator

Blog Navigator 是一个面向技术研究者与内容开发者的结构化博客文章导航与元数据检索系统。该项目对来自技术博客站点的海量文章条目进行统一索引、分类标注与快速访问路径构建，帮助用户在不依赖站内搜索功能的前提下，通过外部结构化目录体系直接定位到具体技术文章页面。

该项目主要服务于需要批量查阅特定领域技术笔记、故障排查记录或代码示例的开发者群体，以及希望将外部博客文章整合进自身知识管理流程的研究人员。通过构建稳定的文章引用列表与分类映射关系，Blog Navigator 将原本分散的博客文章组织为可预测、可遍历的资源网络，从而降低信息重复检索成本，提升技术调研效率。

## 功能概览

**文章条目索引**：对指定博客站点下的文章详情页进行统一编号与条目化处理，每个条目均保留原始 URL 作为永久访问凭证。

**多维度分类视图**：根据文章 ID 区间与主题关键词，将收录的文章划分为开发笔记、问题排查、代码实现、架构设计等基础类别，便于按主题浏览。

**快速跳转定位**：提供完整的文章外部链接列表，用户可直接从项目文档中复制 URL 至浏览器访问，无需进入源站首页逐级导航。

**条目元数据标注**：每条文章链接均附带基础的访问状态与文件类型提示，辅助用户判断内容格式与可用性。

**批量列表导出**：支持将全部索引文章以纯文本列表形式导出，方便用户进行二次筛选或导入到其他书签管理工具。

**轻量化部署**：项目本身为静态 Markdown 文档，无需运行任何后端服务，用户通过本地文件系统或代码仓库即可完成查阅。

## 应用场景

**技术文档撰写前的参考资料收集**：开发者在撰写技术方案或 API 使用说明前，可通过本项目的分类列表快速查阅相关主题的博客文章，获取多种实现思路与问题视角，丰富自身文档的内容深度。

**历史技术笔记归档与追溯**：当博客平台发生内容迁移或结构调整时，本项目提供的静态链接列表可作为历史文章的快照索引，帮助用户持续追踪已有知识条目而不受站点改版影响。

**团队内部技术分享素材池**：技术团队可将本项目作为文章素材池的入口，成员按需从列表中挑选文章进行阅读与讨论，降低团队内部在信息源筛选上的时间损耗。

**个人知识管理系统的外部信源接入**：配合个人维基或笔记软件使用，将本项目中的文章链接批量导入外部知识库，形成以博客文章为节点的个人技术学习图谱。

## 快速开始

以下操作步骤指导用户在本地环境获取并运行 Blog Navigator 项目文档。

```bash
# 克隆项目仓库至本地
git clone https://github.com/example/blog-navigator.git

# 进入项目根目录
cd blog-navigator

# 在默认浏览器中打开项目文档（无需额外安装依赖）
# Linux / macOS
xdg-open README.md
# Windows
start README.md
```

若用户需要使用 Markdown 阅读器或其他文档预览工具，可直接使用系统默认的 Markdown 渲染软件打开 `README.md` 文件。

## 安装要求

本项目为纯文档型仓库，不包含可执行代码或运行时环境，但用户仍需满足以下基本条件以正常使用全部功能。

| 依赖项 | 必需性 | 说明 |
|--------|--------|------|
| Markdown 渲染器 | 必需 | 用于正确显示项目文档结构与列表内容，推荐任意支持标准 CommonMark 的编辑器或阅读器 |
| 文本编辑器 | 必需 | 用于查看或复制文档中的 URL 列表，推荐 VS Code、Sublime Text 或 Notepad++ |
| Git 客户端 | 推荐 | 用于通过克隆命令获取仓库最新版本，若用户仅需阅读文档也可直接下载 ZIP 压缩包 |
| 网络连接 | 必需 | 访问资源列表中的全部外部博客文章页面时必须具备稳定的互联网连接 |
| 主流网页浏览器 | 必需 | 用于打开资源列表中的文章详情页，推荐 Chrome、Firefox 或 Edge 最新版本 |

## 贡献指南

本项目欢迎外部贡献者参与资源列表的扩充、分类优化与文档改进。所有贡献需遵循以下流程。

第一，复刻项目仓库至个人账户，并在本地完成变更。复刻操作可通过 GitHub 页面上的 Fork 按钮完成，随后使用 git clone 命令将复刻后的仓库拉取到本地环境。

第二，在资源列表末尾按原始格式追加新增文章链接，或对现有条目进行勘误与修正。所有 URL 必须保持原样复制，禁止添加协议头、修改域名大小写或补充尾部斜杠。

第三，提交变更前需运行 `scripts/validator.py` 脚本对新增 URL 进行基础格式校验，确保所有链接符合 `http://www.blog.nzfnve.cn/Article/details/` 的路径规范，且不包含非法字符。

第四，提交拉取请求至主仓库的 `main` 分支，并在请求描述中明确说明本次变更的类型（新增条目、分类调整或文档勘误），附上变更所依据的原始数据来源或校对依据。

第五，等待项目维护者审核。审核通过后，变更将被合并至主分支，并同步更新 `CHANGELOG.md` 中的批次记录与更新时间。

## 常见问题

**问：资源列表中的部分链接无法访问，应如何处理？**

答：由于源站内容更新或页面迁移，部分历史文章链接可能出现 404 或访问重定向。用户可尝试复制文章 ID 后在源站首页的站内搜索框中直接检索，或通过 Internet Archive 等网页存档服务回溯历史版本。项目维护者会定期在批次更新时清理长期失效的链接，但无法保证所有外部页面永久可访问。

**问：项目文档是否提供按主题分类的筛选视图？**

答：当前版本的文档以完整列表形式呈现全部资源链接，未在 README 中提供动态分类筛选功能。用户可参考 `docs/navigation/index.md` 中的分类映射说明，根据文章 ID 区间自行归纳主题类别。项目贡献者正在规划基于标签体系的扩展目录，预计在后续批次更新中引入分类分组展示。

**问：如何获取后续批次的资源列表更新？**

答：项目主仓库会在每批次资源整理完成后更新 README.md 文件，并在 CHANGELOG.md 中记录新增批次编号与条目数量。用户可通过定期拉取仓库最新代码或关注仓库的 Release 通知获取更新。当前项目运行批次为第 194 批，后续批次将按计划持续发布。
