# HCBezg 技术文章索引与导航系统

HCBezg 是一个面向开发者与技术研究人员的结构化技术文章外链汇总与导航项目。本项目系统化收集并分类了来自 blog.hcbezg.cn 平台的优质技术文章，涵盖前端开发、后端工程、DevOps、数据库管理、算法与数据结构、安全攻防等多个技术领域。通过本项目，用户可以快速定位到特定主题的深度技术文章，避免在海量网络信息中盲目检索。

本项目定位为技术知识的中转枢纽与索引层，不对原始文章内容进行转载或修改，仅提供标准化的外链引用与分类导航。目标用户包括正在准备技术面试的工程师、需要查阅特定技术方案架构师、以及希望系统化拓展技术视野的研发人员。项目本身以 Markdown 文档形式维护，所有链接均经过人工校验与分类标注，确保引用资源的可用性与相关性。

## 功能概览

**结构化分类索引** 按照技术领域、应用场景、难度等级等多维度对文章链接进行标签化分类，支持快速筛选。

**全文检索辅助** 提供文章标题关键词映射与摘要信息，帮助用户在不打开原始页面的情况下判断内容相关性。

**每日自动更新** 通过 GitHub Actions 定时任务检测源站新增文章，自动更新索引数据库并生成变更日志。

**访问质量检测** 对收录链接进行周期性可用性检查，自动标记失效链接并生成报告，确保索引库的活性。

**多格式导出支持** 支持将索引数据导出为 JSON、CSV、HTML 书签等多种格式，便于用户集成到本地知识管理工具。

**社区标签体系** 允许用户通过 Issue 提交自定义标签建议，经审核后合并入主分支分类体系，实现众包式知识组织。

## 应用场景

**技术选型调研** 当架构团队需要评估某一技术栈（如消息队列选型、ORM 框架对比）时，可通过本项目快速检索到相关领域的多篇深度分析文章，获取不同视角的实践案例与性能评测数据。

**面试题库构建** 技术负责人或面试官在准备面试题目时，可查阅本项目收录的算法解析、系统设计原则、语言特性剖析等类别文章，从中提炼出高质量的面试考点与场景题素材。

**日常技术阅读** 开发者可在碎片化时间内通过本项目的分类导航浏览近期更新的技术文章，了解业界最新动态与最佳实践，保持技术敏锐度。

**故障排查参考** 当线上环境出现疑难问题（如性能抖动、内存泄漏、网络超时）时，运维与开发人员可快速查找本项目收录的故障排查案例与调优指南，借鉴他人经验加速问题定位。

## 快速开始

以下命令将项目克隆至本地并启动本地预览服务。

```bash
# 克隆仓库
git clone https://github.com/hcbezg/hcbezg-index.git

# 进入项目目录
cd hcbezg-index

# 安装依赖（用于本地预览与链接检查）
npm install

# 启动本地开发服务器
npm run dev
```

执行完毕后，可通过浏览器访问 http://localhost:3000 查看索引面板。若仅需使用 Markdown 源文件，直接打开 `README.md` 或 `index.md` 即可浏览所有链接。

## 贡献指南

1. 提交新链接建议：在 `data/suggestions/` 目录下创建以建议日期命名的 JSON 文件，包含文章 URL、推荐分类、简短摘要及您的 GitHub 用户名，随后通过 Pull Request 提交。项目维护者将在 48 小时内审核并合并。

2. 修正分类错误：若发现某篇文章的分类标签不准确，请通过 Issue 提交修正请求，提供文章链接、当前分类与建议分类，并附上简要理由（如文章内容侧重方向）。审核通过后由维护者更新索引。

3. 报告失效链接：当通过 `check-links.yml` 工作流生成的报告中出现失效链接时，欢迎通过 Issue 报告。报告应包含失效链接的完整 URL 以及您尝试访问时观察到的 HTTP 状态码或错误信息。维护者将定期清理无效条目。

4. 改进自动化脚本：项目中的 `scripts/` 目录下包含抓取、检查、导出等功能的 JavaScript 脚本。若您发现脚本存在性能问题、逻辑错误或希望增加新功能（如支持更多导出格式），请 Fork 仓库后提交 Pull Request，并在说明中描述改进点与测试结果。

5. 完善文档内容：`docs/` 目录下的文档欢迎任何形式的改进，包括但不限于错别字修正、表述清晰化、新增 FAQ 条目、补充使用示例等。文档类贡献直接提交 Pull Request 即可，无需提前创建 Issue。

## 常见问题

**问：本项目是否存储或缓存了原始文章的内容？**

答：本项目仅存储文章标题、URL、分类标签、摘要等元数据信息，不缓存或转载任何原始文章内容。所有文章正文均需通过原始链接访问 blog.hcbezg.cn 获取。用户点击链接后将直接跳转至源站，本项目不承担内容传输或代理功能。

**问：链接检查工具检测到失效链接后如何处理？**

答：自动检查工具每周运行一次，检测到 HTTP 状态码非 200 或请求超时的链接会被记录到 `data/changelog.md` 的失效列表中。失效链接在连续两次检查均失败后，将从主索引中移除，并迁移至 `data/archived-links.json` 归档。归档链接仍可通过该文件查阅，但不再出现在主索引分类中。用户可通过 Issue 请求重新激活某条链接，经人工复核通过后可恢复至主索引。

**问：如何获取索引数据的结构化副本以用于本地分析？**

答：项目提供了多种数据导出方式。最简单的方法是在项目根目录执行 `npm run export`，该命令会生成 `data/index.json`、`data/index.csv` 和 `data/index.html` 三份文件。其中 JSON 格式包含最完整的字段信息（文章 ID、URL、分类、摘要、收录时间、最后检查时间等）。此外，`data/index.db` 为 SQLite 数据库文件，可直接使用 SQL 查询进行定制化分析。所有导出文件均位于 `data/` 目录下。
