# TechResources Aggregator

TechResources Aggregator 是一个面向开发者与技术研究人员的开源技术资料聚合与导航系统。该项目系统性地收集、分类并索引互联网上的高质量技术文章、博客内容与工程实践文档，帮助用户快速定位特定主题下的深度技术解读与案例分析。项目定位为技术知识的外链枢纽，不存储任何侵权内容，仅提供公开可访问的互联网资源索引服务。

本项目适用于日常技术研究、架构选型调研、故障排查参考以及技术视野拓展等场景。通过结构化的资源分类与标签体系，用户可以在海量信息中高效筛选出与自身需求匹配的材料，节省重复检索与过滤的时间成本。项目本身以静态站点形式交付，支持本地部署与在线访问两种使用方式，兼顾便捷性与数据掌控力。

## 功能概览

**按主题分类浏览** 资源按照编程语言、框架生态、基础设施、算法与设计模式等主题进行一级分类，方便用户从宏观领域切入检索。

**全文关键词检索** 基于倒排索引实现对资源标题、摘要与标签的快速关键词匹配，支持多词组合查询与高亮定位。

**资源时效性标注** 每条资源均记录收录时间与原始发布时间，支持按时间范围筛选，确保用户获取的信息具有时效参考。

**外链健康检测** 系统定期对收录的 URL 进行可用性探测，自动标记失效链接并生成异常报告，保障导航质量。

**个性化收藏夹** 注册用户可创建自定义收藏列表，将常用资源分组保存，支持标签备注与快速导出。

**导入导出机制** 支持以 JSON 与 CSV 格式批量导入外部资源列表，亦可导出全量索引数据用于离线分析或备份迁移。

**阅读模式适配** 针对文章类资源提供无干扰阅读模板，自动提取正文内容并重构排版，优化移动端与桌面端阅读体验。

## 应用场景

**技术选型调研** 架构师或技术负责人计划引入新的框架或中间件时，可通过本项目的相关分类快速获取多篇实践对比文章与性能评测报告，综合多方观点辅助决策。

**故障排查参考** 开发者在生产环境遇到异常或性能瓶颈时，可利用检索功能查找类似问题的排查记录与解决方案，借鉴他人的处理经验缩短修复时间。

**技术学习路径规划** 初学者或转岗工程师可按主题分类系统性地阅读入门教程与进阶指南，构建完整的知识图谱，避免信息碎片化带来的认知负担。

**文档与博客聚合管理** 技术博主或团队知识管理员可将分散在多个平台的文章链接统一收录至本系统，形成内部知识库的入口索引，便于团队共享与查阅。

## 快速开始

以下指令适用于 Linux / macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/techresources/aggregator.git

# 进入项目目录
cd aggregator

# 安装依赖（基于 Python 3.10 + pip）
pip install -r requirements.txt

# 初始化本地数据库并导入示例资源
python manage.py init_db
python manage.py import_resources --source data/sample_resources.json

# 启动本地开发服务器
python manage.py runserver --host 127.0.0.1 --port 8080
```

服务启动后，访问 http://127.0.0.1:8080 即可浏览资源列表与执行检索操作。生产环境部署请参考 `docs/deployment.md` 中的 Nginx + Gunicorn 配置示例。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 或更高 | 核心运行环境，建议使用 3.10.12 LTS |
| SQLite | 3.35.0 或更高 | 默认嵌入式数据库，用于存储资源索引与用户数据 |

| make | 3.81 或更高 | 用于执行自动化构建脚本（非必需，可手动替代） |

以上版本要求基于主流发行版测试环境制定，若使用较早版本的操作系统，请参考 `docs/compatibility.md` 中的适配方案。

## 常见问题

**问：检索某些关键词时返回结果为空，但预期应该存在相关资源，如何排查？**

答：请首先确认关键词拼写无误。本系统采用基于分词器的倒排索引，若关键词为复合术语或包含特殊符号，建议拆分为更短的核心词进行检索。亦可尝试在高级搜索中切换为模糊匹配模式。此外，检查 `data/resource_index.db` 是否完整导入，若索引文件损坏可执行 `python manage.py rebuild_index` 重建索引。

**问：外链健康检测提示大量链接失效，是否会影响日常浏览？**

答：健康检测结果仅用于参考与维护，不影响资源的展示与检索。失效链接在详情页会有状态标记，但用户仍可手动访问原始地址。项目维护者会定期清理长期失效的链接，但用户也可通过贡献渠道提交更新后的 URL 或替代链接。

**问：本地部署后前端页面样式错乱或检索无响应，可能是什么原因？**

答：此类问题通常由前端资源未正确构建或后端 API 地址配置错误导致。请确认执行了完整的前端构建步骤（`npm run build`），并检查 `frontend/src/config.js` 中的 API_BASE_URL 是否与后端实际监听地址一致。若仍无法解决，请查看浏览器开发者工具的控制台报错信息，并参考 `docs/troubleshooting.md` 中的对应章节。
