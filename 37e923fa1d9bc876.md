# LinkVault 技术资源索引系统

LinkVault 是一个面向开发者、技术研究人员与内容创作者的轻量级技术外链资源汇总与导航系统。该项目定位于解决技术社区中高质量文章、教程与深度分析内容分散、难以检索与长期保存的问题，通过结构化的索引机制和静态站点生成能力，将散落于各类技术博客、专栏与社区中的优质外链进行统一收录、分类、标签化与全文元数据检索。

目标用户包括：需要维护团队技术知识库的架构师、定期整理技术周报的社区运营者、希望构建个人阅读工作流的全栈工程师，以及需要对特定技术领域进行系统性文献梳理的研究人员。LinkVault 不提供内容存储服务，仅对链接及其元数据进行索引与管理，通过标准的 Markdown 配置文件驱动，输出为可直接部署的静态 HTML 页面，便于内网或公网快速发布。

## 功能概览

**批量外链导入** 支持从 CSV、JSON 及 Markdown 列表批量导入原始链接，自动解析 URL、标题与来源域名。

**自动元数据抓取** 在导入阶段自动请求目标页面，提取 HTML 标题、meta description、主要正文前 200 字符及发布日期，作为检索依据。

**自定义标签与分类体系** 允许用户为每条链接打上多级标签，支持基于标签树结构的浏览与过滤，满足不同技术领域的细粒度分类需求。

**全文检索与高级过滤** 内置基于倒排索引的轻量级搜索功能，支持按标题、描述、标签、域名及时间范围进行组合过滤。

**静态站点生成引擎** 将索引数据渲染为响应式 HTML 页面，支持列表视图、卡片视图与详情页三种展示模式，输出产物无需后端服务即可运行。

**定期健康检查** 内置链接有效性检查模块，可配置定时任务对已收录链接进行 HTTP 状态码检测，标记失效或重定向链接，并生成巡检报告。

**数据导入导出** 支持将整个索引库导出为标准 JSON Schema 或 CSV 格式，便于迁移至其他知识管理工具或进行二次分析。

## 应用场景

**团队技术周报自动化** 技术负责人每周收集团队内部推荐的优秀技术文章链接，通过 LinkVault 导入并标记所属方向，系统自动生成带分类索引的周报页面，直接发送给全体研发成员查阅。

**个人技术阅读工作流** 开发者在日常浏览过程中通过浏览器书签或稍后读工具收集待读文章，定期批量导入 LinkVault，利用自动元数据抓取功能生成阅读清单，按标签筛选优先阅读高优先级内容。

**开源项目文档外链管理** 开源项目维护者将项目相关的教程、生态工具、第三方评测报告等外链统一收录至 LinkVault，生成 External Resources 页面并嵌入项目文档站点，方便社区用户快速获取扩展资料。

**技术培训课程资料汇总** 培训机构或企业内训讲师将每期课程涉及的参考文章、视频链接与官方文档地址录入 LinkVault，通过标签区分章节与难度等级，生成课程专属资源导航页，学员可一键访问全部课外阅读材料。

## 快速开始

以下命令演示了从源码克隆到启动开发服务的完整流程，默认使用 SQLite 作为元数据存储，无需额外配置数据库服务。

```bash
git clone https://github.com/linkvault/linkvault.git
cd linkvault
pip install -r requirements.txt
python manage.py migrate
python manage.py import --file samples/links_250.csv
python manage.py runserver
```

访问 http://127.0.0.1:8000 即可查看索引首页。生产环境部署请参考 `docs/deployment.md` 中的 Nginx + Gunicorn 配置示例。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |

| Python | 3.9 及以上 | 核心运行环境，推荐使用 3.11 以获得性能提升 |
| pip | 22.0 及以上 | Python 包管理器，用于安装项目依赖 |
| SQLite | 3.35 及以上 | 默认元数据存储引擎，支持 JSON 字段操作 |

| jinja2 | 3.1.0 及以上 | 模板渲染引擎，用于静态站点生成 |

## 贡献指南

欢迎社区开发者提交贡献，请遵循以下流程以确保代码质量和项目一致性。

1. 查阅 issue 列表，选择未被认领的待办事项或提出新功能建议，在对应 issue 下留言表明认领意向，避免多人重复工作。

2. Fork 本仓库至个人账号，创建以功能名称命名的 feature 分支，例如 `feature/opengraph-support` 或 `fix/csv-encoding-issue`。

3. 编写代码时遵循 PEP 8 编码规范，为新增函数添加 docstring，并为关键逻辑编写对应的单元测试用例，确保测试覆盖率达到 80% 以上。

4. 提交 commit 时使用语义化消息格式：`<type>(<scope>): <subject>`，其中 type 包括 feat/fix/docs/style/refactor/test 等，scope 为影响的模块名称。

5. 发起 Pull Request 至 main 分支，在描述中详细说明改动内容、测试结果以及是否涉及破坏性变更，等待维护者审阅。合并前需通过 CI 流水线的全部检查项。

## 常见问题

Q: 导入包含中文标题的 CSV 文件时出现乱码，应如何解决？

A: 请将 CSV 文件编码转换为 UTF-8 with BOM 格式，或在导入命令中指定 `--encoding gbk` 参数。LinkVault 默认使用 UTF-8 解析，若源文件来自 Windows Excel 保存的 CSV，通常为 GBK 编码，需显式指定编码参数。

Q: 健康检查模块报错 SSL 证书验证失败，如何处理？

A: 部分老旧站点可能使用自签名证书或已过期的证书链。可在配置文件中将 `HEALTH_SSL_VERIFY` 设置为 `false` 以跳过证书验证。请注意，此操作会降低安全性，仅建议在内网环境或明确信任的源站范围内使用。

Q: 静态站点生成后部分链接详情页显示标题为空白，是什么原因？

A: 元数据抓取阶段可能因目标站点反爬策略或网络超时而未能获取到页面标题。可在导入后通过管理命令 `python manage.py refetch --id <link_id>` 单独重新抓取，或在导入时增加 `--timeout 10` 参数延长等待时间。若大量链接存在此问题，建议检查网络出口 IP 是否被目标站点限制。
