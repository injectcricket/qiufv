# LinkVault

LinkVault 是一个面向技术研究者、内容聚合者和开发者的高质量外链资源导航与归档系统。该项目旨在解决技术信息分散、优质文章难以回溯、个人知识库缺乏结构化外链管理等问题，通过将分散在网络各处的技术文章、教程、文档和案例按照统一索引进行归集，帮助用户构建可检索、可追溯、可持续维护的外部知识引用网络。

本项目定位于技术资源的外链汇总站，不直接托管内容，而是提供清晰的目录结构、分类标签和快速检索能力，使用户能够高效管理和复用来自特定源站的大量技术资料链接。LinkVault 适用于个人博客知识库建设、团队技术文档外链整理、开源项目参考资料归档等场景。

## 功能概览

- **批量链接导入与解析**：支持从文本文件、CSV 或直接粘贴的 URL 列表中批量导入链接，自动解析 HTTP 状态码、响应头信息，并提取页面标题和摘要描述。

- **分类标签与全文检索**：每个外链可绑定多个自定义标签，基于 SQLite FTS5 全文搜索引擎，支持对链接标题、描述、标签和备注字段进行毫秒级检索。

- **链接健康状态监控**：定期对已收录的链接发起 HEAD 请求，检测死链、重定向链和证书过期问题，并在仪表盘中以颜色标记状态（正常、警告、失效）。

- **元数据自动补全**：通过 Open Graph 协议和页面 Meta 标签自动抓取链接的标题、描述、作者信息和发布日期，减少手动录入成本。

- **层级目录树管理**：提供无限层级的目录分类结构，用户可以将链接按照技术栈、项目阶段或知识领域进行树状组织，支持拖拽调整节点位置。

- **访问统计与热度排序**：记录每个链接的点击次数、最后访问时间和收藏次数，支持按热度、时间或自定义权重进行排序展示。

- **数据导入导出**：支持将整个链接库导出为 JSON、CSV 或 HTML 书签格式，也支持从 Firefox/Chrome 书签导出文件进行增量合并。

- **RSS 订阅源生成**：每个目录节点可生成独立的 RSS 订阅地址，方便团队内部或订阅者实时获取新增链接动态。

## 应用场景

**技术博客外链库管理**：独立技术博主在撰写文章时引用了大量外部资料，使用 LinkVault 可以统一存放这些引用链接，并按照文章主题分类。当需要更新某篇旧文章的外部链接或验证引用源是否仍可访问时，通过检索功能快速定位，大幅提升内容维护效率。

**开源项目参考文档聚合**：开源项目维护者需要整理相关的设计文档、API 参考、社区讨论帖和第三方集成示例。LinkVault 允许为每个项目版本建立独立的链接目录，将分散在多个平台的高质量参考链接集中管理，并可在项目 README 中直接引用生成的链接列表。

**技术团队知识库外链整合**：企业技术团队在内部 Wiki 或文档系统中引用了大量外部技术文章，但这些引用往往散落各处且缺乏统一维护。LinkVault 可作为团队知识库的外部链接补充层，由专人负责链接的准入审核、分类标注和健康监控，确保团队文档中的外部引用始终可用且符合安全策略。

**个人技能学习路线规划**：学习者围绕某一技术方向（如 Kubernetes 运维、Rust 后端开发）收集了大量教程和案例链接。LinkVault 的层级目录和标签系统可以帮助学习者按照学习阶段（入门、进阶、实战）和子领域（网络、存储、调度）对链接进行二维组织，形成清晰的学习地图。

## 快速开始

以下指令可在任何安装了 Python 3.10+ 和 Git 的环境中完成 LinkVault 的部署与启动。

```bash
# 克隆代码仓库
git clone https://github.com/example/linkvault.git
cd linkvault

# 创建 Python 虚拟环境并激活
python3 -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate

# 安装项目依赖
pip install -r requirements.txt

# 初始化 SQLite 数据库并创建表结构
python scripts/init_db.py

# 导入示例链接数据（包含批次 223/280 的样本链接）
python scripts/import_links.py --source data/sample_links.csv

# 启动 Web 服务（默认监听 127.0.0.1:8000）
python app.py
```

访问本地地址 http://127.0.0.1:8000 即可进入 LinkVault 仪表盘。首次启动时系统会自动创建一个管理员账户，登录信息将在终端输出中显示，请及时修改默认密码。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.10 或更高 | 核心运行环境，低于此版本将无法使用异步特性及类型注解 |
| SQLite3 | 3.35.0 或更高 | 内置数据库引擎，需支持 FTS5 全文索引扩展 |
| pip | 22.0 或更高 | Python 包管理器，用于安装依赖项 |

| lxml | 4.9.0 或更高 | 高性能 XML/HTML 解析器，作为 beautifulsoup4 的后端 |
| flask | 2.2.0 或更高 | Web 框架，提供 HTTP 接口和模板渲染 |
| flask-login | 0.6.0 或更高 | 用户会话管理扩展 |
| apscheduler | 3.10.0 或更高 | 后台任务调度器，用于定时执行链接健康检查 |
| pytz | 2022.0 或更高 | 时区处理库，支持计划任务的时区配置 |

## 贡献指南

LinkVault 是一个社区驱动的开源项目，欢迎任何形式的贡献。请遵循以下步骤参与开发：

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境。确保本地 Python 版本满足 3.10 以上要求，同时安装好所有开发依赖（可使用 `pip install -r requirements-dev.txt` 安装额外测试工具）。

2. 创建新的功能分支，分支命名遵循 `feature/功能简述` 或 `fix/问题简述` 格式。在提交代码之前，请先运行 `pytest tests/` 确保所有现有测试用例通过，并为新增功能补充对应的测试代码。

3. 代码风格遵循 PEP 8 规范，使用 Black 格式化工具统一代码风格（`black app/ scripts/ tests/`）。提交信息采用约定式提交格式，例如 `feat: 添加批量标签更新接口` 或 `fix: 修复健康检查超时导致的僵尸进程问题`。

4. 完成开发后，向本仓库的主分支提交 Pull Request。PR 描述中需清晰说明变更目的、影响范围及测试情况。项目维护者会在 3 个工作日内进行 Code Review，并给出合并或修改意见。

5. 若发现安全漏洞或严重缺陷，请直接通过项目维护者的安全邮箱报告，不要提交公开 Issue，以便在修复后统一发布补丁版本。

## 常见问题

**问：LinkVault 是否支持 HTTPS 协议的源站链接？**

答：完全支持。LinkVault 的链接抓取和健康检查模块基于 requests 库，默认跟随重定向并兼容 HTTPS 协议。如果源站证书无效或过期，健康检查会将其标记为「警告」状态，但不会阻止链接收录。用户可以在配置文件中设置 `SSL_VERIFY = False` 关闭证书验证，但不建议在生产环境中使用。

**问：批量导入 250 条以上的链接时，系统会不会变慢或超时？**

答：系统针对批量导入做了事务优化和异步化处理。每 50 条链接提交一次数据库事务，同时元数据抓取采用异步协程并发执行，默认并发数为 10。对于 250 条链接，完整导入（含元数据补全）通常在 30 至 60 秒内完成。如果网络环境较差，可以在导入命令中添加 `--no-fetch` 参数，仅导入链接 URL，后续再通过后台任务逐步补全元数据。

**问：如何从其他书签工具迁移到 LinkVault？**

答：LinkVault 提供了 `scripts/import_links.py` 脚本，支持导入 Chrome/Firefox 导出的 HTML 书签文件、Netscape 格式书签以及通用 CSV 格式。对于浏览器书签，系统会自动解析书签文件夹路径并映射为目录分类。执行命令示例：`python scripts/import_links.py --source bookmarks.html --type browser`。迁移后原有的文件夹层级将被完整保留。
