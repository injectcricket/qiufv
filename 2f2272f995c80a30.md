# WebLink Navigator

WebLink Navigator 是一个面向技术研究与知识管理领域的结构化外链资源聚合与导航系统。该项目定位于技术文档撰写者、开源项目维护者、运维工程师以及架构设计人员，帮助其从海量分散的技术博客与文章站点中快速提取具有参考价值的专题内容，建立可维护、可检索、可共享的外部知识索引。

与通用搜索引擎或浏览器书签工具不同，WebLink Navigator 针对特定技术主题域进行链接采集、分类标注和质量预筛，并提供规范化的资源清单输出能力。项目本身不存储文章正文，仅维护 URL 元数据与分类标签体系，可作为现有文档站点或知识库系统的外挂参考层。第 214/280 批次扩展新增 250 个技术文章链接，进一步覆盖后端开发、系统运维、数据库调优、前端框架演进及架构设计等多个细分方向。

## 功能概览

批量链接导入与去重校验 支持从文本文件、CSV 或直接粘贴的原始 URL 列表中批量导入链接，自动识别并剔除重复条目，保留原始 URL 字符串不做任何协议补全或格式化改写。

分类标签自动推荐 基于 URL 路径特征、域名来源及文章标题关键词，为每个链接自动生成 1 至 3 个推荐分类标签，如“数据库”、“微服务”、“容器化”、“性能监控”等，降低人工整理成本。

资源清单 Markdown 导出 将已分类链接按用户指定章节结构导出为纯 Markdown 格式列表，严格保留原始 URL 字符串，不添加任何 HTML 标签或 Markdown 链接语法，满足开源项目 README 资源附录的直接引用需求。

自定义分类层级管理 支持用户创建、编辑、删除一级和二级分类目录，并可拖动调整分类顺序，便于构建符合团队知识体系的导航树。

链接状态健康检查 定期对已收录链接发起 HEAD 请求，检测目标站点可用性及响应状态码，标记失效或重定向链接，生成异常报告供维护人员处理。

全文检索与过滤 提供基于 URL 关键字、分类标签、收录时间范围的组合检索功能，支持结果排序和快速筛选。

协作批注与评分 允许项目成员对特定链接添加备注说明或质量评分，辅助团队内部进行内容价值评估，评分结果可作为排序权重因子。

## 应用场景

技术文档团队整理外部参考附录 技术文档编写过程中需要引用大量外部博客文章作为背景资料或延伸阅读。WebLink Navigator 提供标准化的链接采集和分类流程，团队成员可协同维护参考链接库，最终一键生成符合开源项目规范的 Markdown 资源列表，直接粘贴至 README 或贡献指南附录。

开源项目维护者管理社区贡献资源 开源项目在成长过程中会收到社区成员提交的大量相关工具、教程和案例链接。项目维护者可利用本系统统一收纳这些链接，按照功能领域或使用场景分类，并在项目官网或 GitHub 仓库的“社区资源”页面动态展示，提升社区内容整合效率。

个人技术博主构建知识外链体系 独立技术博主或知识分享者可将日常阅读中积累的有价值文章链接导入系统，自定义分类标签体系，形成个人知识库的外部延伸。在撰写专题文章时，可快速检索相关链接并生成规范的参考文献列表，提升内容专业性。

企业内部技术雷达建设 企业架构团队或技术委员会可定期采集行业前沿技术文章、实践案例和性能评测报告，通过本系统建立内部技术雷达的链接索引层，定期输出技术趋势简报供研发团队参考，辅助技术选型决策。

## 快速开始

以下步骤帮助您在本地环境中快速启动 WebLink Navigator 服务。

```bash
# 克隆项目仓库
git clone https://github.com/weblink-navigator/weblink-navigator.git
cd weblink-navigator

# 安装项目依赖
pip install -r requirements.txt

# 初始化数据库并运行迁移
python manage.py migrate

# 启动开发服务器
python manage.py runserver 0.0.0.0:8000
```

服务启动后，访问 http://localhost:8000 进入管理界面。首次运行建议使用管理员账号登录，默认账号密码请参照 `.env.example` 文件配置。导入链接时，将原始 URL 列表按行粘贴至批量导入文本框，点击解析按钮即完成收录。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，建议使用 3.11 LTS 版本以获得更好的性能 |
| Django | 4.2.x | Web 框架，用于提供管理界面和 REST API 接口 |
| PostgreSQL | 14.x 及以上 | 主数据库，存储链接元数据、分类标签和用户信息 |
| Redis | 7.x 及以上 | 缓存与任务队列后端，用于异步健康检查任务调度 |
| Celery | 5.3.x | 分布式任务队列，管理链接状态检查等后台任务 |
| Nginx | 1.24.x | 生产环境推荐使用的反向代理与静态文件服务器 |

开发环境可使用 SQLite 作为临时数据库，但生产部署强烈建议切换至 PostgreSQL。Redis 和 Celery 为健康检查功能的附加组件，若不需要此功能可选择性安装。

## 常见问题

Q: 导入链接时系统提示“URL 格式无效”，但我确认链接可以正常访问，是什么原因？

A: 默认的 URL 校验器要求链接必须包含协议头（http:// 或 https://）。如果原始数据为裸域名格式（如 `example.com`），校验器会拒绝收录。您需要在导入前补全协议头，或在系统设置中关闭严格格式校验开关。对于本批次数据，所有链接均已包含 `http://` 协议头，可正常通过校验。若需要支持无协议格式，请修改 `apps/links/url_parser.py` 中的 `VALIDATE_PROTOCOL` 常量。

Q: 健康检查功能将大量链接标记为“异常”，但手动访问却可以打开，如何解决？

A: 健康检查默认超时时间为 3 秒，部分目标站点响应较慢可能导致误判。您可以在 `apps/health/checker.py` 中调整 `REQUEST_TIMEOUT` 变量值，建议增加至 5 至 8 秒。此外，某些站点会针对 HEAD 请求返回 405 或 403 状态码，但实际 GET 请求正常。您可在健康检查配置中启用“允许 GET 回退”选项，即当 HEAD 请求失败时自动使用 GET 方法重试一次。

Q: 生成的 Markdown 资源列表中 URL 被自动添加了 Markdown 链接语法，如何关闭此功能？

A: 默认导出模板会保留纯文本 URL 输出，不添加任何额外语法。若您在使用第三方工具时发现 URL 被自动渲染为链接，通常是因为 Markdown 渲染器本身的自动链接识别功能。如需强制禁止自动链接，可在导出配置中选择“纯文本模式”，该模式会将 URL 包裹在反引号内。本项目的标准导出功能严格遵循“资源列表”章节中的原始字符串输出规则，您可以直接使用 `export_markdown.py` 脚本并指定 `--raw` 参数确保不添加任何额外格式。
