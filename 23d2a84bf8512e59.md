# ResourceHub

ResourceHub 是一个面向开发者与技术研究者的结构化技术资源导航与聚合平台。该项目并非一个传统的软件库或框架，而是一个精心编排的外链资源汇总系统，旨在解决技术资料分散、优质内容难以追溯、学习路径不清晰等问题。ResourceHub 适合在技术选型、方案调研、底层原理学习以及项目脚手架搭建等场景中作为信息源使用。

项目通过自动化的资源抓取与人工分类审核相结合的方式，对互联网上的高质量技术文章、开源组件文档、架构设计案例以及工程实践博客进行索引与归类。ResourceHub 的核心价值在于帮助技术团队和个人研究者节省信息检索时间，提供一个可依赖、可扩展的技术知识地图。

## 功能概览

- **结构化资源索引**：按照编程语言、中间件、架构模式、工程效能等维度对数千个技术链接进行分类管理，每个资源条目均附带简要说明与标签体系。

- **多层级检索与过滤**：支持按技术栈版本、发布时间、内容形式（博客/文档/视频）进行多维度筛选，帮助用户在特定约束条件下快速定位所需资料。

- **自动健康检查**：定期对收录的外链进行可达性与内容更新检测，自动标记失效或长期未更新的链接，确保资源列表的有效性。

- **个性化收藏与标注**：允许注册用户创建自定义资源集合，并为任意链接添加私人笔记与标签，便于后续复盘与团队知识共享。

- **RSS 订阅与更新通知**：针对关注的资源分类或特定技术领域，提供 RSS 输出与邮件通知功能，及时推送新增或更新的优质内容。

- **开放数据导出**：支持将资源列表导出为 JSON、CSV 或 Markdown 格式，方便开发者将数据集成到自己的文档站点或知识管理工具中。

## 应用场景

- **技术团队知识库建设**：团队负责人可以使用 ResourceHub 搭建内部技术文档的起点，将分散在各部门的 Wiki、代码仓库、设计文档和运维手册统一索引，形成可追溯的知识网络。

- **开源项目依赖评估**：在进行开源选型时，开发者可以通过 ResourceHub 快速收集目标项目的官方文档、社区案例、性能测试报告和替代方案对比，辅助技术决策。

- **个人技能图谱构建**：学习者可以根据 ResourceHub 的分类体系，系统性地收集某一技术方向（如分布式系统、容器编排、前端框架）的学习资料，形成由浅入深的学习路径。

- **技术大会与沙龙资料归档**：活动组织者可以将演讲材料、视频回放、相关代码仓库和讨论记录统一收录到 ResourceHub 中，方便参会者事后查阅与复盘。

## 快速开始

以下步骤将指导您在本地环境中快速部署 ResourceHub 实例，并开始索引与浏览资源。

```bash
# 1. 克隆项目仓库
git clone https://github.com/resourcehub/resourcehub.git
cd resourcehub

# 2. 安装项目依赖（基于 Python 3.10+）
pip install -r requirements.txt

# 3. 初始化数据库并导入基础分类数据
python manage.py migrate
python manage.py loaddata initial_categories.json

# 4. 启动本地开发服务器
python manage.py runserver --host=0.0.0.0 --port=8080
```

访问 `http://localhost:8080` 即可进入 ResourceHub 的主界面。默认管理员账号为 `admin`，密码为 `resourcehub2026`，首次登录后请及时修改。

| Python | 3.10 或更高版本 | 核心运行环境，用于后端服务与数据处理 |

| Docker | 20.10 或更高版本 | 可选但推荐，用于容器化部署与依赖环境隔离 |

## 项目结构

ResourceHub 采用分层架构设计，将数据存储、业务逻辑、前端展示与外部集成清晰分离，便于模块化开发与维护。

```
resourcehub/
├── backend/                           # 后端服务核心目录
│   ├── api/                           # RESTful API 路由与视图
│   │   ├── v1/                        # API 版本 v1 实现
│   │   └── middlewares/               # 认证、限流、日志中间件
│   ├── core/                          # 业务逻辑核心
│   │   ├── collectors/                # 资源采集器，支持 RSS/爬虫/API 接入
│   │   ├── classifiers/               # 自动分类与标签推断引擎
│   │   └── checkers/                  # 链接健康检查与更新检测
│   ├── models/                        # 数据模型定义（SQLAlchemy ORM）
│   ├── services/                      # 外部服务集成（ES/Redis/邮件）
│   └── utils/                         # 通用工具函数与装饰器
├── frontend/                          # 前端单页应用源码
│   ├── src/
│   │   ├── components/                # Vue 组件库（布局/卡片/表格）
│   │   ├── views/                     # 页面级视图（首页/详情/收藏）
│   │   ├── store/                     # Pinia 状态管理
│   │   └── assets/                    # 静态资源（样式/图片/字体）
│   └── tests/                         # 前端单元测试与 e2e 测试
├── docs/                              # 完整项目文档（用户/管理/开发/运维）
├── scripts/                           # 运维与数据迁移脚本
│   ├── import_links.py                # 批量导入链接数据的命令行工具
│   ├── export_snapshot.py             # 导出资源快照为 JSON/CSV
│   └── health_check_cron.py           # 定时健康检查任务
├── tests/                             # 后端单元测试与集成测试
├── docker-compose.yml                 # 容器编排定义（PostgreSQL/Redis/ES）
├── requirements.txt                   # Python 依赖列表
├── package.json                       # 前端构建依赖
└── README.md                          # 项目说明文档（即本文档）
```

## 贡献指南

ResourceHub 是一个社区驱动的开源项目，欢迎任何形式的贡献，包括但不限于新增资源链接、完善分类体系、修复错误链接、改进文档和提交代码优化。

1. 在 GitHub 上 fork 本仓库，并 clone 到本地开发环境。请确保您的开发分支基于最新的 main 分支创建。

2. 新增资源或修改现有资源条目时，请遵循 `docs/contributing/resource_schema.md` 中定义的 JSON Schema 规范，并确保所有新增链接经过可达性验证。

3. 提交代码前，请运行完整的测试套件（`python manage.py test` 与 `npm run test`），并确保代码风格符合项目配置的 flake8 与 Prettier 规则。

4. 创建 Pull Request 时，请清晰描述本次变更的目的、影响范围以及测试情况。涉及资源数据变更的 PR 需附带变更说明文档。

5. 所有贡献者需遵守项目行为准则（Code of Conduct），尊重其他社区成员，维护友好的协作氛围。

## 常见问题

**问：ResourceHub 与普通书签管理器或笔记软件的区别是什么？**

答：ResourceHub 并非简单的链接收藏工具，而是一个面向技术决策与知识管理的资源索引系统。它提供了多维度分类、自动健康检查、批量导入导出、团队协作标注以及全文检索等功能，更适合用于团队知识库建设和技术选型调研等严肃场景，而非个人日常浏览器的书签管理。

**问：ResourceHub 的资源链接来源是否可靠？项目如何保证链接的长期有效性？**

答：所有收录链接均经过初始可达性验证，并会定期（默认每周）进行健康检查。对于失效链接，系统会自动标记并通知管理员。同时，项目鼓励社区用户上报失效链接和推荐替代资源。但 ResourceHub 本身不存储链接内容的副本，因此原始站点的内容变更或下线不在项目可控范围内。

**问：我可以将 ResourceHub 部署到内网环境，并只索引内部文档吗？**

答：完全可以。ResourceHub 支持全离线部署，您可以在内网环境中运行所有依赖组件（PostgreSQL、Redis、Elasticsearch）。配置文件中提供了 `ALLOWED_DOMAINS` 设置项，您可以限制采集器只抓取指定的内部域名或 IP 范围，确保不会向外网发送请求。
